# RobotMaster对抗赛雷达功能模块 - README.md
## 项目概述
RobotMaster对抗赛雷达功能模块是为RobotMaster机甲大师对抗赛定制开发的核心感知组件，旨在为参赛机甲提供**实时战场环境感知、敌方目标定位、障碍物检测、距离测算**等关键能力，是机甲实现自主决策、精准打击和避障防御的核心基础。

该模块基于毫米波雷达/激光雷达（可适配赛事规则允许的传感器）硬件，结合嵌入式算法与上位机数据解析，实现对战场内多目标的实时扫描、坐标解算、速度/方向识别，并与机甲的运动控制、武器系统联动，最终提升机甲在对抗中的态势感知与战术执行效率。

## 核心功能
| 功能项                | 详细说明                                                                 |
|-----------------------|--------------------------------------------------------------------------|
| 多目标实时检测        | 同时识别战场内敌方机甲、掩体、场地边界等多个目标，刷新率≥20Hz（适配赛事要求） |
| 目标坐标解算          | 将雷达原始数据转换为战场绝对坐标/相对坐标，输出目标(x,y,θ)位置信息          |
| 距离与角度测算        | 精准计算机甲与目标的直线距离（误差≤5cm）、相对角度，为武器瞄准提供数据支撑 |
| 目标动态追踪          | 对移动目标（敌方机甲）进行持续追踪，输出目标运动速度、方向等动态特征       |
| 数据轻量化传输        | 对雷达数据进行滤波、降噪、压缩，适配机甲串口/以太网通信带宽，降低延迟     |
| 异常数据容错处理      | 针对雷达干扰、遮挡等场景做鲁棒性处理，保障数据输出稳定性                 |

## 技术栈
### 硬件适配
- 雷达传感器：支持毫米波雷达（如TI AWR1843）、激光雷达（如思岚A1/A2）（可根据赛事规则替换）
- 主控板：STM32H743/ESP32（嵌入式数据预处理）、NVIDIA Jetson Nano（上位机数据解析）
- 通信模块：串口（UART）、以太网、CAN总线（适配机甲总线协议）

### 软件/算法
- 嵌入式开发：C/C++、STM32 HAL库
- 上位机开发：Python/C++、OpenCV（数据可视化）、ROS（可选，适配机甲系统）
- 核心算法：卡尔曼滤波（数据降噪）、目标聚类（多目标区分）、坐标变换（雷达坐标系→战场坐标系）

## 快速开始
### 环境依赖
#### 嵌入式端（STM32）
- 开发工具：Keil MDK 5.36+、STM32CubeMX
- 依赖库：STM32 HAL库、雷达传感器驱动库（如TI mmWave SDK）

#### 上位机端（PC/ Jetson）
- 操作系统：Ubuntu 20.04（推荐）/Windows 10
- 依赖包：
  ```bash
  # Python依赖
  pip install pyserial numpy matplotlib opencv-python
  # C++依赖（可选）
  sudo apt install cmake libeigen3-dev libserial-dev
  ```

### 部署步骤
1. **硬件连接**
   - 将雷达传感器通过SPI/UART接口连接至STM32主控板；
   - 主控板通过以太网/CAN总线连接至机甲上位机；
   - 供电并检查硬件连接稳定性（推荐使用稳压电源）。

2. **嵌入式程序烧录**
   - 打开`STM32_Radar_Project/MDK-ARM/Radar.uvprojx`工程；
   - 根据使用的雷达型号修改`radar_config.h`中的参数（波特率、采样率等）；
   - 编译并烧录至STM32主控板。

3. **上位机数据解析**
   - 运行上位机解析脚本：
     ```bash
     python radar_parser.py --port /dev/ttyUSB0 --baudrate 115200
     ```
   - 脚本会实时解析雷达数据，输出目标坐标、距离等信息，并提供可视化界面。

## 调试与常见问题
### 调试工具
- 串口调试助手：验证雷达原始数据是否正常输出；
- 示波器：检查雷达供电电压、通信信号稳定性；
- 上位机可视化工具：实时查看目标检测效果，定位数据异常点。

### 常见问题解决
| 问题现象                  | 可能原因                          | 解决方案                                  |
|---------------------------|-----------------------------------|-------------------------------------------|
| 雷达无数据输出            | 供电异常/串口波特率不匹配         | 检查供电电压（5V/12V）、核对`radar_config.h`波特率 |
| 目标坐标误差大            | 坐标系未校准/雷达安装角度偏移     | 重新校准雷达安装角度，更新坐标变换参数    |
| 多目标识别混乱            | 滤波算法参数不合理/环境干扰       | 调整卡尔曼滤波参数，增加目标聚类阈值      |
| 数据传输延迟高            | 串口波特率过低/数据未压缩         | 提升波特率（≤115200），启用数据压缩逻辑  |

## 赛事适配建议
1. 根据赛事规则调整雷达参数：如赛事限制雷达扫描频率，需在`radar_config.h`中降低采样率；
2. 适配机甲通信协议：若机甲采用ROS系统，可将解析后的数据封装为ROS话题（参考`PC_Parser/ros_adapter.py`）；
3. 轻量化优化：对抗赛中需降低CPU占用率，可关闭非必要的可视化功能，仅保留核心数据输出；
4. 冗余设计：备用一套雷达传感器及代码，应对赛场硬件故障。
